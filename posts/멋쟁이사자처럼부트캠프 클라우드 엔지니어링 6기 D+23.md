<h3 id="1-웹-아키텍처와-정적-컴포넌트html의-처리">1. 웹 아키텍처와 정적 컴포넌트(HTML)의 처리</h3>
<p>웹 서비스는 크게 자원을 요청하는 클라이언트와 요청에 응답하는 서버의 구조로 나뉜다. HTML은 이 과정에서 서버에서 연산 없이 전달되는 <strong>정적 컴포넌트(Static Component)</strong> 역할을 수행한다.</p>
<ul>
<li><strong>서버와 클라이언트의 역할:</strong><ul>
<li><strong>Server:</strong> 클라이언트의 요구사항을 서비스해주는 주체(Apache, Nginx 등 웹 서버 및 DB 서버).</li>
<li><strong>Client:</strong> 서버에게 필요한 작업을 요청하는 주체(웹 브라우저).</li>
</ul>
</li>
<li><strong>정적 자원의 통신 흐름:</strong><ol>
<li><strong>요청(Request):</strong> 브라우저가 서버에 특정 HTML 파일을 요청한다.</li>
<li><strong>검색:</strong> 서버는 해당 파일을 찾는다. 파일이 없으면 404 오류(File Not Found)를 발생시킨다.</li>
<li><strong>응답(Response):</strong> 파일이 존재하면 HTML과 연결된 CSS, JS, 이미지 등을 클라이언트로 전송한다.</li>
<li><strong>캐싱(Caching):</strong> 브라우저는 다운로드한 자원을 로컬 캐시 저장소에 저장한다.</li>
<li><strong>렌더링:</strong> 브라우저 엔진이 파일을 실행하여 화면에 출력한다.</li>
</ol>
</li>
<li><strong>캐시의 효율성:</strong> 이전 요청과 동일한 자원을 요구할 경우, 서버에 다시 접속하지 않고 로컬 캐시에서 가져와 출력 속도를 높인다.</li>
</ul>
<blockquote>
<p><strong>Note:</strong> 정적 컴포넌트는 서버에서 로직이 실행되지 않으므로 항상 동일한 화면을 렌더링한다. 반면 JSP와 같은 동적 컴포넌트는 서버의 자바 코드가 실행된 후 결과값만 HTML로 응답한다.</p>
</blockquote>

<h3 id="2-브라우저-렌더링의-근간-dom-구조와-특징">2. 브라우저 렌더링의 근간: DOM 구조와 특징</h3>
<p>DOM(Document Object Model)은 브라우저가 HTML 문서를 해석하여 메모리에 생성한 트리 자료구조이며 모든 노드는 객체로 존재한다.</p>
<ul>
<li><strong>트리 구조와 노드의 독립성:</strong><ul>
<li>최상위 <code>Document</code> 노드 아래에 모든 태그가 계층적으로 연결된다.</li>
<li>태그(Element), 속성(Attribute), 텍스트(Text)는 각각 <strong>독립적인 객체(Node)</strong>로 존재하며, 이를 통해 특정 텍스트 노드만 수정하는 등 조작이 가능하다.</li>
</ul>
</li>
<li><strong>노드 간의 계층적 관계:</strong><ul>
<li><strong>부모-자식 (Parent-Child):</strong> 직계 상하 관계.</li>
<li><strong>형제 (Sibling):</strong> 같은 부모를 둔 나란한 관계의 노드들이다.</li>
<li><strong>조상-자손 (Ancestor-Descendant):</strong> 직계를 넘어선 모든 상위/하위 포함 관계를 의미한다.</li>
</ul>
</li>
</ul>

<h3 id="6-form과-전송-방식get-vs-post">6. Form과 전송 방식(GET vs POST)</h3>
<p>사용자 입력 데이터를 서버로 전달하는 시스템이다.</p>
<ul>
<li><strong>데이터 조립:</strong> <code>submit</code> 발생 시 브라우저는 <code>name</code> 속성을 Key로, 입력값을 Value로 매핑하여 쿼리 스트링을 만든다.</li>
<li><strong>전송 방식 비교:</strong><ul>
<li><strong>GET :</strong> 데이터가 URL 끝에 Query String 형태로 포함된다. 데이터가 주소창에 노출되어 보안이 필요 없는 '조회'에 적합하하다.</li>
<li><strong>POST:</strong> 데이터가 은닉되어 전송된다. URI에 노출되지 않아 상대적으로 안전하며, 대용량 데이터나 서버 상태 변경(생성/수정)에 적합하다.</li>
</ul>
</li>
</ul>
