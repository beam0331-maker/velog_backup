<h1 id="til-java-제어문과-배열">[TIL] Java 제어문과 배열</h1>
<h2 id="1-문장의-종류">1. 문장의 종류</h2>
<p>자바의 문장은 실행 방식에 따라 <strong>실행문</strong>과 <strong>비실행문(주석)</strong>으로 나뉜다. </p>
<h3 id="실행문">실행문</h3>
<ul>
<li><strong>순차문</strong> : 순차적으로 실행되는 문장, <code>;</code>으로 끝난다.</li>
<li><strong>제어문</strong> : 조건식에 따라서 실행이 달라지는 문장으로 <strong>조건문</strong>, <strong>반복문</strong>이 있다.</li>
</ul>
<hr />
<h2 id="2-조건문-if-와-switch">2. 조건문: if 와 switch</h2>
<h3 id="1-단일-if문">(1) 단일 if문</h3>
<ul>
<li><p><strong>용도</strong>: 특정 조건이 참(<code>true</code>)일 때만 코드를 실행하고자 할 때 사용한다.</p>
<pre><code class="language-java">  if(조건식){ 
      // 조건식이 true인 경우 실행될 내용
  }</code></pre>
</li>
</ul>
<h3 id="2-if-else문">(2) if-else문</h3>
<ul>
<li><p><strong>용도</strong>: 조건의 <code>true</code>와 <code>false</code>에 따라 실행할 내용을 나누고 싶을 때 사용한다.</p>
<pre><code class="language-java">  if(조건식){ 
      // 조건식이 true인 경우 실행될 내용
  } else {
      // 조건식이 false인 경우 실행될 내용
  }</code></pre>
</li>
</ul>
<h3 id="3-다중-if문-if-else-if문">(3) 다중 if문 (if-else if문)</h3>
<ul>
<li><p><strong>용도</strong>: 조건이 여러 개일때 사용한다. 각 조건에 따라 다른 실행이 필요할 때 사용하며 상단 조건이 만족되면 하단의 조건은 검사하지 않는다.</p>
<pre><code class="language-java">  if(조건식1){ 
      // 조건식1이 true인 경우 실행될 내용
  } else if(조건식2){
      // 조건식1은 false이고 조건식2가 true인 경우 실행될 내용
  } else if(조건식3){
      // 조건식1,2이 false이고 조건식3이 true인 경우 실행될 내용
  }else{
      // 위의 모든 조건이 false인 경우 실행될 내용
  }</code></pre>
</li>
</ul>
<h3 id="4-switch문">(4) switch문</h3>
<ul>
<li><strong>용도</strong>: 변수의 '값'에 따라 여러 경로 중 하나를 선택해야 할 때 사용한다. (<code>byte</code>, <code>short</code>, <code>int</code>, <code>char</code>, <code>String</code>, <code>enum</code> 사용 가능)</li>
</ul>
<h4 id="기존-switch문">기존 switch문</h4>
<ul>
<li><code>break</code>가 없으면 다음 <code>case</code>까지 실행한다. 일반적으론 <code>break</code>를 통해 <code>case</code>를 구분하다. 모든 <code>case</code>에 해당하지 않으면 <code>default</code>가 실행된다.</li>
<li><strong>선언 예시</strong>:<pre><code class="language-java">  switch(변수){
      case 값1:
          // 변수 == 값1일 때 실행
          break;
      case 값2:
          // 변수 == 값2일 때 실행
          break;
      default:
          // 일치하는 값이 없을 때 실행
  }</code></pre>
</li>
</ul>
<h4 id="switch-expression-jdk-14">Switch Expression (JDK 14+)</h4>
<ul>
<li><code>-&gt;</code> 사용하며, <code>break</code>가 없어도 각 <code>case</code>의 내용을 실행하며 결과를 변수에 대입할 수 있다,</li>
<li><strong>선언 예시</strong>:<pre><code class="language-java">  int result = switch(n) {
      case 10 -&gt; 100;    // n = 10 이라면 100을 result에 반환
      case 20 -&gt; 200;    // n = 20 이라면 200을 result에 반환
      default -&gt; 0;
  };</code></pre>
</li>
</ul>
<hr />
<h2 id="3-반복문">3. 반복문</h2>
<p>반복문은 필수정보 <strong>초기값, 조건식, 증감식</strong>에 의해서 작동한다.</p>
<h3 id="1-for문">(1) for문</h3>
<ul>
<li><strong>주요 사용 상황</strong>: 반복 횟수가 명확하게 정해져 있는 경우에 주로 사용한다. (예: 배열 인덱스)</li>
<li><strong>실행 흐름</strong>: <code>초기값 → 조건식 → (true) 냉용 실행 → 증감식→ 조건식</code></li>
<li><strong>예시</strong>:<pre><code class="language-java">  for(int i = 0; i &lt; 10; i++){ // for(초기값;조건식;증감식)
      // i가 0부터 9까지 총 10번 반복 실행
  }</code></pre>
</li>
</ul>
<h3 id="2-while문">(2) while문</h3>
<ul>
<li><strong>주요 사용 상황</strong>: 무한루프를 유도할때 주로 사용된다.</li>
<li><strong>실행 흐름</strong>: <code>조건식 검사 → (true) 내용 실행 → 다시 조건식 검사</code></li>
<li><strong>예시</strong>:<pre><code class="language-java">  int i = 0; // 초기값
  while(조건식) {    // 조건식
       // 조건식이 true인 동안 반복 실행
       // 증감식은 내용에 포함되어있다.
  }</code></pre>
</li>
</ul>
<h3 id="3-do-while문">(3) do-while문</h3>
<ul>
<li><strong>주요 사용 상황</strong>: 반복 조건과 상관없이 <strong>최소 1회 실행</strong>이 반드시 보장되어야 하는 경우에 사용한다.</li>
<li><strong>실행 흐름</strong>: <code>내용 실행(최초 1회) → 조건식 검사 → (true)내용 실행</code></li>
<li><strong>예시</strong>:<pre><code class="language-java">  int i = 0 // 초기값
  do {
      // 일단 한 번 실행하고, 이후 조건식이 true이면 반복
      // 증감식이 내용에 포함되어있다.
  } while(조건식);</code></pre>
</li>
</ul>
<h3 id="break--continue">break &amp; continue</h3>
<ul>
<li><strong>break</strong>: 반복문을 즉시 탈출하여 종료한다.</li>
<li><strong>continue</strong>: 남은 코드를 스킵하고 즉시 다음 루틴(증감식 또는 조건식)으로 넘어간다.</li>
</ul>

<h2 id="4-배열array">4. 배열(Array)</h2>
<p>동일한 타입의 여러 데이터를 하나의 변수로 관리하는 구조이다.</p>
<pre><code class="language-java">// 배열선언
데이터형[] 변수명
// 배열생성
변수명 = new 데이터형[크기]
// 일반적인 예시
int[] n = new int[10];
</code></pre>
<h3 id="메모리-구조">메모리 구조</h3>
<ul>
<li>배열 변수 자체는 <strong>Stack</strong>에 생성되며, 실제 배열의 데이터가 저장된 <strong>Heap의 주소값</strong>을 가진다.</li>
<li><strong>자동 초기화</strong>: 배열은 Heap 영역에 생성되는 <strong>인스턴스</strong>이므로, 객체의 인스턴스 변수와 마찬가지로 생성 시 타입별 기본값으로 자동 초기화된다.</li>
</ul>

<ul>

<li>한 번 생성된 배열은 <strong>크기 변경이 불가능</strong>하다.</li>

<h3 id="오늘의-회고">오늘의 회고</h3>
<p>오늘의 회고 어제와 같이 블로그를 작성하며 호기심에 찾아보게 된것들을 적어 볼까한다.</p>
<h4 id="1--이란">1. <code>[]</code> 이란</h4>
<ul>
<li>배열 선언 시 사용하는 <code>[]</code>는 해당 변수가 스택(Stack)에 실제 값이 아닌 힙(Heap) 영역의 주소를 저장하는 <strong>참조 변수</strong>임을 정의하는 표식이다.</li>
<li>관례상 <code>타입[] 변수명</code> 형태를 권장하며, 이는 '타입'과 '데이터 구조'를 명확히 분리하여 코드의 가독성을 높여준다.</li>
<li>힙 영역에 생성된 실제 데이터 덩어리(인스턴스)에 접근하기 위한 유일한 통로로서, 메모리 효율성과 대규모 데이터 관리의 핵심 역할을 수행한다.</li>
</ul>
<h4 id="2-개선된-for문-enhanced-for--jdk-15">2. 개선된 for문 (Enhanced for / JDK 1.5+)</h4>
<ul>
<li><p>향상된 switch를 찾아보다 찾게 되었다.</p>
</li>
<li><p><strong>지원 버전</strong>: <strong>JDK 1.5</strong> 이상부터 사용 가능하다.</p>
</li>
<li><p><strong>특징</strong>: 배열을 순회할 때 인덱스를 직접 관리하지 않아 코드가 간결하고 인덱스 관련 실수(IndexOutOfBounds)를 방지할 수 있다.</p>
</li>
<li><p><strong>예시</strong>:</p>
<pre><code class="language-java">  // 기존 for문
  int[] scores = {90, 80, 100};
  for(int i = 0; i &lt; scores.length; i++) {
      System.out.println(scores[i]);
  }

  // 개선된 for문 (JDK 1.5+)
  for(int score : scores) {
      System.out.println(score); // 배열의 요소를 직접 꺼낼수 있다.
  }</code></pre>
<p> 위 내용들과 더불어 배열의 자동초기화는 코딩을 하며 당연히 알고 있었던 내용있어지만, 어제 배운 인스터스 변수와 같이 메모리 영역의 이론이었다는게 신기하면서 문명 예전엔 배웠던 내용이었을텐데 너무 쉽게 까먹었던게 아닐까란 생각이 들었다.</p>
</li>
</ul>
