<h3 id="1-자바의-데이터-저장-방식-3가지-비교">1. 자바의 데이터 저장 방식 3가지 비교</h3>
<p>데이터를 담는 그릇은 크게 3가지로 나눌 수 있다.</p>
<ul>
<li><strong>변수:</strong> 하나의 값만 저장할 수 있다. 데이터 개수만큼 변수가 필요해서 데이터가 많아지면 관리가 너무 힘들어진다.</li>
<li><strong>배열:</strong> 여러 개의 데이터를 저장 가능하며 데이터 개수와 무관하게 배열명 하나만 필요해 관리가 용이하다. </li>
<li><strong>컬렉션:</strong> 저장되는 데이터형이 달라도 저장이 가능하고 자동으로 크기가 늘어난다. 컬렉션 자체는 참조형만 저장 가능하여 기본형 데이터는 Wrapper 클래스(<code>Integer</code> 등)를 통해 참조형의 형태로 저장된다.</li>
</ul>
<h3 id="2-배열-유틸리티-javautilarrays">2. 배열 유틸리티: <code>java.util.Arrays</code></h3>
<p>배열을 다룰 때, 복잡한 로직 없이 자바에서 제공하는 유틸리티를 사용할 수 있다.</p>
<p><strong>① 배열 출력하기: <code>Arrays.toString()</code></strong>
배열을 문자열로 모두 출력해 준다.</p>
<p><strong>② 배열 채우기: <code>Arrays.fill()</code></strong></p>
<pre><code class="language-java">String[] name = { &quot;A&quot;, &quot;B&quot;, &quot;C&quot;, &quot;2&quot;, &quot;3&quot;, &quot;F&quot;, &quot;T&quot; };
Arrays.fill(name, 0, 3, &quot;X&quot;); // 0번부터 3번 '전'까지 &quot;X&quot;로 변경 (끝 위치 미포함)</code></pre>
<p><strong>③ 배열 정렬: <code>Arrays.sort()</code></strong>
배열을 오름차순으로 정렬한다. (내림차순이나 객체 정렬은 주말에 더 파헤쳐볼 예정이다!)</p>
<p><strong>④ 특정 값의 인덱스 찾기: <code>Arrays.binarySearch()</code></strong></p>
<ul>
<li><strong>🤔 회고 포인트:</strong> 왜 <code>binarySearch</code>는 반드시 <strong><code>sort()</code>가 전제</strong>되어야 하는가.
이진 탐색은 중간 값을 기준으로, 찾는 값보다 크면 아래쪽 절반은 버리고 윗부분만 검색한다. 즉 정렬된 데이터를 전제로 진행되는 검색이므로 <code>sort()</code>가 반드시 전제되어야 한다.</li>
</ul>
<p><strong>⑤ 배열 비교하기: <code>Arrays.equals()</code></strong>
배열 안의 요소들이 같은지 비교한다.</p>
<p><strong>⑥ 배열 크기 변경 및 복사: <code>Arrays.copyOf()</code></strong>
크기가 고정된 배열의 단점을 극복하기 위해, 새로운 배열을 만들어서 기존 배열값을 복사해 넣는 방식으로 자주 쓰인다.</p>
<hr />
<h3 id="3-향상된-for문-enhanced-for">3. 향상된 for문 (Enhanced for)</h3>
<p>어제 다른 제어문을 찾아보다가 <strong>'향상된 for문(JDK 1.5+)'</strong>을 짧게 언급한 적이 있었다. 오늘은 향상된 for문을 통해 얻는 장점을 알 수 있었다.</p>
<ul>
<li><strong>선언 방식 및 비교</strong><pre><code class="language-java">int[] numbers = {10, 20, 30, 40, 50};
</code></pre>
</li>
</ul>
<p>// 1. 기존 for문 (인덱스 사용)
for (int i = 0; i &lt; numbers.length; i++) {
    System.out.println(numbers[i]);
}</p>
<p>// 2. 향상된 for문 (인덱스 없이 요소만 순회)
for (int num : numbers) {
    System.out.println(num);
}</p>
<p>```</p>
<ul>
<li><strong>💡 최대 장점: <code>ArrayIndexOutOfBoundsException</code> 원천 차단</strong>
기존 <code>for</code>문은 조건식에서 <code>&lt;= numbers.length</code>처럼 등호 하나만 잘못 써도 인덱스 범위를 벗어나고 <code>ArrayIndexOutOfBoundsException</code> 에러가 발생한다. 
하지만 <code>향상된 for</code>문은 배열의 처음부터 끝까지만 호출하고 멈추기 때문에, <strong>인덱스 계산 실수로 인한 에러를 원천적으로 방지</strong>할 수 있어 코드가 훨씬 안전하게 작동하게 된다.</li>
</ul>
<hr />
<h3 id="📝-회고_딥다이브-배열-복사의-깊이-차이">📝 회고_딥다이브: 배열 복사의 '깊이' 차이</h3>
<p>어제 정리했던 Heap 영역과 참조 변수는 오늘 배열에서도 중요하게 작용했다.</p>
<p><strong>1) 얕은 복사 (Shallow Copy)</strong>
<code>int[] arr2 = arr1;</code>처럼 대입 연산자(<code>=</code>)를 쓰면 데이터 자체가 아니라 <code>Heap 영역</code>의 <strong>'주소값'</strong>만 복사된다. 결국 두 변수가 같은 메모리를 바라보게 되어 하나를 수정하면 다른 하나도 변하게 된다. <code>Stack</code> 영역의 변수 복사와 혼동하면 치명적인 오류가 야기될 수도 있다.</p>
<p><strong>2) 깊은 복사 (Deep Copy)</strong>
<code>Heap 영역</code>에 새로운 메모리 공간을 확보한 뒤, 원본 데이터의 내용을 복사하는 방식이다. <code>배열.clone()</code>, <code>Arrays.copyOf()</code>를 사용하면 깊은 복사를 쉽게 구현할 수 있다. 원본과 다른 메모리를 가지고 있기 때문에 수정해도 원본에는 영향을 끼치지 않는다.</p>
<p><strong>3) 2차원 이상의 깊은 복사 (Deep Copy)</strong>
새로운 메모리를 만드는 <code>clone()</code>이나 <code>copyOf()</code>만으로는 2차원 이상의 배열에서 완벽한 깊은 복사가 불가능하다. 완벽한 복사를 위해서는 하위 배열까지 <code>for</code>문을 통해 일일이 복사해야 한다.</p>

<p>오늘 강의는 배열과 Arrays에 대한 내용이 주를 이루었다. 강의 내용을 그대로 포스팅하는 게 글이 과하게 길어지는 것 같아서 새로운 양식을 적용해 보았다. 강의 내용을 개인의 중요도에 따라 재단하는 게 맞나 싶지만... 어쩔 수가 있나, 내 벨로그인걸!</p>
