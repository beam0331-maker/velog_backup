<h3 id="1-sql-언어의-기능별-분류와-실행-메커니즘">1. SQL 언어의 기능별 분류와 실행 메커니즘</h3>
<ul>
<li><strong>[개념 및 특징]</strong>
SQL은 처리 목적에 따라 DDL, DML, TCL로 분류된다.</li>
<li><strong>[분류]</strong><ul>
<li><strong>DML (Data Manipulation Language):</strong> 행(Row) 단위의 데이터 변경을 수행하며, 명시적 커밋 전까지는 임시 상태로 유지된다.</li>
<li><strong>DDL (Data Definition Language):</strong> 데이터 객체 구조를 결정하며, 실행 즉시 <strong>암시적 커밋</strong>이 발생하여 복구가 불가능하다.</li>
<li><strong>TCL (Transaction Control Language):</strong> 논리적 작업 단위를 확정하거나 취소하여 데이터 일관성을 제어한다.</li>
</ul>
</li>
</ul>
<h3 id="2-트랜잭션transaction-제어와-오토커밋-설정">2. 트랜잭션(Transaction) 제어와 오토커밋 설정</h3>
<ul>
<li><strong>[개념 및 특징]</strong>
트랜잭션은 데이터베이스의 상태를 변화시키는 논리적 단위이다. 일부 쿼리만 성공하여 발생하는 <strong>데이터 무결성 훼손</strong>을 방지하기 위해 'All or Nothing' 원칙을 준수한다.</li>
<li><strong>[세션 환경 설정 및 확인]</strong>
MySQL는 기본적으로 오토커밋이 활성화되어 있으나, 세션 단위로 제어할 수 있다.</li>
</ul>
<pre><code class="language-sql">-- 현재 세션의 오토커밋 설정 상태 확인
SHOW VARIABLES LIKE 'autocommit%';

-- 오토커밋 비활성화 (명시적 트랜잭션 모드 전환)
SET autocommit = false;

-- [실습 시나리오] 계좌 이체 처리
-- 출금과 입금이 하나의 논리적 단위로 묶여 트랜잭션 불일치를 방지함
UPDATE accounts SET balance = balance - 500 WHERE account_name = 'A';
UPDATE accounts SET balance = balance + 500 WHERE account_name = 'B';

-- 모든 연산이 정상일 경우 물리적 DB에 확정 반영
COMMIT; 
-- 논리적 오류 발생 시 작업 전 상태로 원복
-- ROLLBACK;</code></pre>
<h3 id="3-데이터-조작어dml-선언과-집합-단위-데이터-처리">3. 데이터 조작어(DML) 선언과 집합 단위 데이터 처리</h3>
<ul>
<li><strong>[개념 및 특징]</strong>
테이블의 레코드를 관리하는 명령어다. 대량의 데이터를 처리할 때는 <code>VALUES</code>를 반복하기보다 <code>SELECT</code> 결과셋을 직접 삽입하여 처리 효율을 높인다.</li>
</ul>
<pre><code class="language-sql">-- 1. INSERT INTO ... SELECT (집합 단위 데이터 이관)
-- 타 테이블의 특정 조건 데이터를 백업 테이블로 일괄 삽입함
INSERT INTO member_backup (member_id, member_name, salary)
SELECT member_id, member_name, salary
FROM member_table
WHERE region = 'Seoul';

-- 2. 다중행 삽입 (Multi-row INSERT)
INSERT INTO department (dept_no, dname)
VALUES (50, '개발팀'), (60, '보안팀'), (70, '기획팀');

-- 3. UPDATE / DELETE (WHERE절 조건 명시 필수)
-- 조건 미지정 시 의도치 않은 전체 데이터 수정을 초래함
UPDATE member_table SET salary = salary * 1.1 WHERE dept_no = 10;
DELETE FROM member_table WHERE resign_date IS NOT NULL;</code></pre>
<h3 id="4-데이터-정의어ddl---테이블-생성과-스키마-복제">4. 데이터 정의어(DDL) - 테이블 생성과 스키마 복제</h3>
<ul>
<li><strong>[개념 및 특징]</strong>
데이터 저장 구조를 정의하는 명령어로, 실행 시점에 커밋이 실행된다. 기존 테이블의 구조를 복제할 수 있다.</li>
</ul>
<pre><code class="language-sql">/*
CREATE TABLE [IF NOT EXISTS] 테이블명 (
    컬럼명1 데이터타입 [DEFAULT 기본값] [제약조건],
    컬럼명2 데이터타입 [NOT NULL],
    [CONSTRAINT 제약조건명] PRIMARY KEY (컬럼명)
);
*/

-- [CTAS] 원본 테이블의 스키마 구조만 복제 (데이터 제외)
-- WHERE 1=2 조건으로 False를 유도하여 구조만 추출함
CREATE TABLE member_schema_copy AS 
SELECT * FROM member_table 
WHERE 1=2; </code></pre>
<h3 id="5-외래-키foreign-key-참조-무결성">5. 외래 키(Foreign Key) 참조 무결성</h3>
<ul>
<li><strong>[개념 및 특징]</strong>
자식 테이블이 부모 테이블의 기본 키(PK)를 참조하도록 강제하여 <strong>참조 무결성</strong>을 보장한다. 부모 데이터 삭제 시 옵션을 통해 <strong>참조 무결성</strong>을 유지한다.</li>
</ul>
<table>
<thead>
<tr>
<th align="left">옵션</th>
<th align="left">물리적 동작 및 영향</th>
</tr>
</thead>
<tbody><tr>
<td align="left"><strong>CASCADE</strong></td>
<td align="left">부모 레코드 삭제 시 참조 중인 자식 레코드도 연쇄적으로 삭제함.</td>
</tr>
<tr>
<td align="left"><strong>SET NULL</strong></td>
<td align="left">부모 레코드 삭제 시 자식의 외래 키 컬럼을 NULL로 업데이트함.</td>
</tr>
<tr>
<td align="left"><strong>RESTRICT</strong></td>
<td align="left">자식 레코드가 존재하는 경우 부모 레코드의 삭제 동작을 차단함. (기본값)</td>
</tr>
</tbody></table>
<pre><code class="language-sql">-- 참조 무결성 제약 조건 및 삭제 전이 정책 선언
CREATE TABLE order_list (
    order_id INT PRIMARY KEY,
    member_id INT,
    -- 부모 데이터 제거 시 자식 데이터의 무결성 훼손을 막기 위한 정책 설정
    CONSTRAINT fk_member_id FOREIGN KEY (member_id) 
    REFERENCES member_table (member_id) 
    ON DELETE SET NULL 
);</code></pre>
<h3 id="6-alter-drop-truncate">6. ALTER, DROP, TRUNCATE</h3>
<ul>
<li><strong>[개념 및 특징]</strong>
객체의 구조를 변경하거나 삭제한다. 특히 데이터 전체를 삭제할 때 <code>DELETE</code>와 <code>TRUNCATE</code>의 차이를 정확히 알고 있어야한다.</li>
</ul>
<blockquote>
<p>🌟 ** DELETE와 TRUNCATE의 결정적 차이**
<code>DELETE</code>는 DML로서 실행 후 롤백이 가능하고 언두(Undo) 로그를 남기지만, <code>TRUNCATE</code>는 DDL로서 자동 커밋되므로 실행 후 복구가 불가능하다.</p>
</blockquote>
<pre><code class="language-sql">-- 1. ALTER: 스키마 구조 변경 (컬럼 추가/삭제)
ALTER TABLE member_table ADD COLUMN phone VARCHAR(20);
ALTER TABLE member_table DROP COLUMN phone;

-- 2. DROP: 객체 소멸 (메타데이터 및 데이터 전체 제거)
DROP TABLE IF EXISTS old_backup;

-- 3. TRUNCATE: 데이터 초기화 (DDL 기반의 고속 삭제)
-- 트랜잭션 로그를 최소화하여 빠른 초기화가 가능하나 롤백이 불가함
TRUNCATE TABLE access_log;</code></pre>
