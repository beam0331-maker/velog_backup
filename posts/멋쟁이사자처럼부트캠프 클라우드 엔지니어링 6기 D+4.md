<h3 id="part-1-데이터-저장-방식-기본형-vs-참조형">Part 1. 데이터 저장 방식: 기본형 vs 참조형</h3>
<p>자바에서 변수는 데이터를 저장하는 방식에 따라 크게 두 가지로 분류된다.</p>
<ul>
<li><strong>기본형 변수 (Primitive Type)</strong>:<ul>
<li><strong>종류</strong>: 정수(<code>byte</code>, <code>short</code>, <code>int</code>, <code>long</code>), 실수(<code>float</code>, <code>double</code>), 문자(<code>char</code>), 논리(<code>boolean</code>) 총 8개입니다.</li>
<li><strong>저장 방식</strong>: 변수 선언으로 확보된 공간에 <strong>실제 값</strong>이 저장된다.</li>
<li><code>num = 10;</code>에서는 해당 메모리 공간에 숫자 10이 직접 저장된다.</li>
</ul>
</li>
<li><strong>참조형 변수 (Reference Type)</strong>:<ul>
<li><strong>종류</strong>: 클래스, 배열, 인터페이스 등</li>
<li><strong>저장 방식</strong>: 변수 공간에는 실제 데이터가 아닌, 데이터가 저장된 메모리의 <strong>주소값(Address)</strong>이 저장된다.</li>
<li>모든 참조형 변수의 크기는 <strong>4byte</strong>로 동일하다.</li>
</ul>
</li>
</ul>

<h3 id="part-3-상수와-데이터-관리">Part 3. 상수와 데이터 관리</h3>
<p><strong>상수 (Constant)</strong></p>
<ul>
<li><strong>선언</strong>:<code>final</code> 키워드를 사용하여 선언</li>
<li>한 번 저장된 값은 변경할 수 없다.</li>
<li>변수와 구별하기 위해 이름을 <strong>대문자</strong>로 지정한다.</li>
<li><strong>용도</strong>: 요일, 학년, 역할(Role) 등 고정된 값을 정의할 때 사용한다.</li>
</ul>
<blockquote>
<p><strong>final</strong>의 확장 <strong>Enum (열거형)</strong></p>
<ul>
<li>상수들을 그룹화하여 관리가능, JDK 1.5부터 도입되었다.</li>
<li><strong>선언</strong>: <code>enum 이름 { 상수1, 상수2 }</code></li>
<li><strong>장점</strong>:  상수를 그룹화 하여 가독성이 좋고, <strong>타입안전성</strong>을 보장받을 수 있다.</li>
</ul>
</blockquote>

<h3 id="part-4-데이터-형-변환">Part 4. 데이터 형 변환</h3>
<p>서로 다른 타입의 데이터가 프로그램 실행중 연산을 위해 타입을 일치시키는 과정.</p>
<ul>
<li><strong>자동 형 변환 (묵시적 형 변환)</strong>: 작은 타입에서 큰 타입으로 대입될 때 자동으로 발생합니다. (<code>int</code> → <code>double</code> 등)<ul>
<li><code>byte</code> &gt; <code>short</code>/<code>char</code> &gt; <code>int</code> &gt; <code>long</code> &gt; <code>float</code> &gt; <code>double</code></li>
<li><code>int</code>보다 작은 타입(<code>byte</code>, <code>short</code>)의 연산 결과는 <code>int</code>로 형 변환된다</li>
<li>작은타입과 큰타입이 연산하면 결과가 큰 타입으로 반환된다.
ex. 정수 + 실수의 결과는 실수 형으로 저장된다.</li>
<li>문자열과 (비)문자열의 합은 문자열로 반환된다
ex. <code>&quot;hello&quot;</code> + <code>10</code> = <code>hello10</code></li>
</ul>
</li>
<li><strong>강제 형 변환 (명시적 형 변환)</strong>: 큰 타입을 작은 타입으로 표현할때 <code>(타입)</code>를 사용한다.</li>
</ul>

<h4 id="2-증감-연산자-전치-vs-후치"><strong>2) 증감 연산자 (전치 vs 후치)</strong></h4>
<p>임의의 변수값을 1씩 증가, 감소시킨다.</p>
<p><strong>[테스트 코드 및 결과]</strong></p>
<ul>
<li><strong>전치</strong>: <code>++변수명</code>, <code>--변수명</code></li>
<li><strong>후치</strong>: <code>변수명++</code>, <code>변수명--</code></li>
</ul>
<pre><code class="language-java">// 1. 전치 (Prefix): 변수값을 먼저 1 증가시킨 후 연산에 참여
int n = 0;
int result1 = ++n; 
System.out.println(result1 + &quot;\t&quot; + n);
// 출력 결과: 1    1

// 2. 후치 (Postfix): 현재 값을 연산에 먼저 사용한 후 변수값을 1 증가
int n2 = 10;
int result2 = n2++; 
System.out.println(result2 + &quot;\t&quot; + n2); 
// 출력 결과: 10    11</code></pre>

<h4 id="인스턴스-변수와-클래스-변수의-메모리-영역차이">인스턴스 변수와 클래스 변수의 메모리 영역차이</h4>
<ul>
<li><strong>인스턴스 변수</strong>: 객체를 생성할 때마다 <code>Heap</code> 영역에 각각 할당되며, 주소를 알아야만 찾아갈 수 있다</li>
<li><strong>클래스 변수</strong>: 프로그램이 실행될 때(클래스가 로딩될 때) 딱 한 번 <strong>메서드 영역(Method Area)</strong>의 Static 영역에 할당된다.</li>
<li>따라서 클래스변수는 특정 개체의 주소를 몰라도 클래스 이름으로 고정된 주소로 접근 가능한 것이다.</li>
</ul>
<h4 id="final의-불안정성"><code>final</code>의 불안정성</h4>
<p><code>public static final int</code>에서 발생하는 문제점이 <code>enum</code>으로 이어진다</p>
<ul>
<li>정수형 상수는 '요일'과 '계절'처럼 서로 다른 의미를 가진 데이터라도 값(예: 1)이 같으면 서로 비교가 가능해지는 논리적 오류가 발생할 수 있다.</li>
<li>Enum은 상수를 각각의 타입로 취급하여 같은 타입으로 비교되면서 발생하는 오류를 해결 할 수 있다.</li>
</ul>
