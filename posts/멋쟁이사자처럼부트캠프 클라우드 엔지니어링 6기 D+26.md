<h3 id="1-javascript-객체json-핵심-문법과-직렬화">1. JavaScript 객체(JSON) 핵심 문법과 직렬화</h3>
<p>객체는 <code>{key: value}</code> 형태의 데이터 구조이며 모든 데이터 타입(수치, 문자열, 배열, 함수)을 값으로 가질 수 있다.</p>
<ul>
<li><strong>접근 방법:</strong> 도트 표기법(<code>obj.name</code>)이 기본이나, 연관 배열 표기법(<code>obj[&quot;name&quot;]</code>)을 사용하면 키(Key)를 사용하여 동적으로 접근할 수 있다.</li>
<li><strong>직렬화와 역직렬화:</strong> 객체를 문자열로, 혹은 그 반대로 변환하는 과정이다.<ul>
<li><code>JSON.parse()</code>: JSON 문자열을 메모리 상의 실제 JS 객체로 파싱한다 (역직렬화).</li>

<li><strong>참고:</strong> 엄격한 JSON 표준에서는 반드시 키와 문자열 값에 큰따옴표(<code>&quot;&quot;</code>)를 써야 파싱 에러를 막을 수 있다.<pre><code class="language-javascript">// 1. 객체를 전송 가능한 JSON 문자열로 직렬화

### 2. 데이터 복사와 병합: 스프레드(Spread) 연산자
스프레드 연산자(`...`)는 배열이나 객체의 요소를 낱개로 펼치는 문법이다.
* **불변성(Immutability) 유지:** 배열을 변수에 단순히 대입하면 주소만 복사되지만, 스프레드 연산자를 사용하면 원본과 독립적인 새로운 주소를 가진 객체가 생성된다.
* **활용:** 안전한 배열 복사, 새로운 요소 추가, 두 배열의 병합에 적극 사용된다.
```javascript
const a = [1, 2, 3];

// 복사: 원본과 메모리 주소가 완전히 분리된 새 배열을 생성한다.
const b = [...a]; 

// 추가: 기존 요소를 펼친 뒤, 새로운 요소를 조합하여 새 배열을 만든다.
const c = [...a, 4, 5]; 

// 연결: 두 배열의 요소들을 하나의 새 배열로 합친다.
const d = [...a, ...b]; </code></pre><h3 id="3-함수-생성-시점-파라미터-1급-객체">3. 함수: 생성 시점, 파라미터, 1급 객체</h3>

### 4. 이벤트(Event) 처리 메커니즘
이벤트는 클릭, 타이핑 등 시스템 동작을 의미하며, 이벤트 태그와 이벤트 핸들러(함수)를 연결하는 작업이다.

| 분류 | 특징 및 문법 | 주의점 |
| :--- | :--- | :--- |
| **인라인 방식** | `&lt;button onclick=&quot;fun()&quot;&gt;` | HTML과 JS 코드가 결합되어 유지보수가 어렵다. |
| **DOM Level 0 (고전)** | `btn.onclick = fun;` | 동일 이벤트에 대해 단 하나의 핸들러만 덮어쓰기로 연결된다. |
| **DOM Level 2 (표준)** | `btn.addEventListener(&quot;click&quot;, fun);` | 이벤트 타입명에 `on`을 빼야 하며, 리스너를 다중 등록할 수 있다. |

&gt;  고전 방식이나 리스너 등록 시 `btn.onclick = fun();`처럼 괄호를 쓰면 함수가 즉시 실행되기 때문에 괄호 없이 함수명(`fun`)만 할당

* **window.onload:** 브라우저가 DOM을 만들기 전에 스크립트가 먼저 실행되면 `null` 에러가 발생한다. 스크립트를 하단에 두거나 `window.onload`를 통해 DOM 트리가 모두 작성된 후에 이벤트를 연결하도록 해야한다.
* **이벤트 전파 차단:** 부모와 자식 태그 모두에 이벤트가 있을 때 자식을 클릭하면 부모 이벤트까지 동작하는 현상이 생긴다. 자식 핸들러 내에서 `event.stopPropagation()`을 호출하여 방지할 수 있다.

<h3 id="6-구조-분해-할당destructuring의-응용">6. 구조 분해 할당(Destructuring)의 응용</h3>
<p>배열이나 객체(JSON) 내부의 값을 짧은 코드로 변수에 매핑하는 기법이다.</p>
<ul>
<li>객체 분해 시에는 순서와 무관하게 키(Key) 이름이 매핑 기준이 된다.</li>
<li>실무 활용: 키 이름 변경(별칭 지정: <code>name: userName</code>), 요소 누락에 대비한 디폴트 값 설정, 임시 변수 없이 배열의 값 뒤바꾸기(Swap: <code>[a, b] = [b, a]</code>), 쉼표(<code>,</code>)를 활용해 특정 요소 건너뛰기 등 다양한 최적화 패턴이 지원된다.</li>
</ul>

<h4 id="1-스프레드-연산자는-완벽한-깊은-복사deep-copy를-보장하는가">1. 스프레드 연산자(...)는 완벽한 깊은 복사(Deep Copy)를 보장하는가?</h4>
<p>스프레드 연산자는 오직 '1차원 깊이'까지만 새로운 메모리를 할당하는 <strong>얕은 복사(Shallow Copy)</strong>로 동작한다.</p>
<p>객체 내부에 또 다른 객체나 배열이 들어있다면, 해당 데이터는 새로운 값이 복사되는 것이 아니라 메모리 주소를 그대로 가져오게 된다. 즉, 복사된 객체 내부의 참조변수를 수정하면 원본 객체의 데이터까지 훼손된다.</p>
<pre><code class="language-javascript">// [얕은 복사(Shallow Copy)의 치명적 한계]
const original = { name: &quot;홍길동&quot;, address: { city: &quot;서울&quot; } };
const shallowCopied = { ...original };

shallowCopied.name = &quot;이순신&quot;; // 1차원 속성은 독립적으로 변경됨 (정상)
shallowCopied.address.city = &quot;부산&quot;; // 중첩 객체(2뎁스) 수정 시 원본도 함께 훼손됨!

console.log(original.address.city); // &quot;부산&quot; 출력 (원본 데이터가 오염됨)

// [깊은 복사(Deep Copy)를 통한 해결책]
const deepCopied = structuredClone(original); // JS 내장 깊은 복사 함수
deepCopied.address.city = &quot;제주&quot;;

console.log(original.address.city); // 원본은 변경되지 않고 그대로 유지됨</code></pre>
<h4 id="2-dom-상태-읽기-innerhtml-vs-innertext-vs-value의-구조적-차이">2. DOM 상태 읽기: <code>innerHTML</code> vs <code>innerText</code> vs <code>value</code>의 구조적 차이</h4>
<p>이벤트를 처리하고 DOM 요소를 조작할 때 대상의 값을 읽어오는 방식이 태그에 따라 달라진다. 제대로 구분하지 않으면 렌더링 에러가 발생한다.</p>
<ul>
<li><code>innerHTML</code>: 일반적인 태그 내부에 접근할 때 사용하며, 텍스트뿐만 아니라 포함된 HTML 조각(예: <code>&lt;strong&gt;</code> 등)까지 모조리 파싱한다. 동적으로 태그를 삽입할 때 유용하지만, XSS(크로스 사이트 스크립팅) 공격에 취약하다</li>
<li><code>innerText</code>: 브라우저가 화면에 실제로 렌더링한 '텍스트 문자열'만 추출한다. 사용자가 보는 글자 그대로를 가져올 때 쓴다.</li>
<li><code>value</code>: 사용자가 타이핑한 값을 가져오기 위해서는 시작/종료 태그 사이가 아닌 <code>value</code>를 호출해야 한다.</li>
</ul>
