<h3 id="1-css-적용-방식">1. CSS 적용 방식</h3>
<pre><code class="language-html">&lt;!DOCTYPE html&gt;
&lt;html&gt;
&lt;head&gt;
  &lt;link rel=&quot;stylesheet&quot; href=&quot;style.css&quot;&gt;

  &lt;style&gt;
    .highlight-text { color: green; }
  &lt;/style&gt;
&lt;/head&gt;
&lt;body&gt;
  &lt;p style=&quot;color: blue;&quot;&gt;Direct Style&lt;/p&gt;
&lt;/body&gt;
&lt;/html&gt;</code></pre>
<ul>
<li><strong>적용 방식별 특징</strong><ul>
<li><strong>Inline Style</strong>: 우선순위가 가장 높다. 디자인이 HTML에 작성되어 유지보수성이 낮음.</li>
<li><strong>Internal Style</strong>: 간단한 페이지 구성시 효율적.</li>
<li><strong>External Style</strong>: <strong>(가장 추천되는 방식).</strong> 디자인을 외부에서 확인, 수정 가능하여 유지보수성이 높다.</li>
</ul>
</li>
</ul>
<hr />
<h3 id="2-캐스캐이딩-알고리즘과-명시도specificity-점수-계산">2. 캐스캐이딩 알고리즘과 명시도(Specificity) 점수 계산</h3>
<ul>
<li><p>여러 스타일 규칙이 충돌할 때 우선순위를 결정하는 폭포수(Cascading) 원칙과 가중치 시스템</p>
</li>
<li><p><strong>캐스캐이딩의 핵심 요소</strong></p>
<ul>
<li><strong>상속(Inheritance)</strong>: 부모의 스타일이 자식으로 전달됨. 하위에서 재정의(Override) 가능.</li>
<li><strong>명시도(Specificity)</strong>: 선택자가 구체적일수록 높은 가중치를 가짐.</li>
<li><strong>소스 순서(Source Order)</strong>: 가중치가 같으면 나중에 선언된 규칙이 적용됨.</li>
</ul>
</li>
<li><p><strong>명시도 계산법 (Specificity Score)</strong></p>
<ul>
<li>인라인 스타일: (1, 0, 0, 0) = 1000점</li>
<li>ID 선택자: (0, 1, 0, 0) = 100점</li>
<li>Class, 속성, 의사 클래스: (0, 0, 1, 0) = 10점</li>
<li>태그 선택자, 의사 요소: (0, 0, 0, 1) = 1점</li>
</ul>
</li>
</ul>
<pre><code class="language-html">&lt;style&gt;
  /* 태그 선택자 (1점) */
  h1 { color: blue; }

  /* 아이디 선택자 (100점) */
  #main-section { color: green; }

  /* 아이디 + 태그 결합 선택자 (101점) -&gt; 최우선순위 */
  #main-section h1 { color: red; }
&lt;/style&gt;
&lt;div id=&quot;main-section&quot;&gt;
  &lt;h1&gt;명시도 우선순위 적용&lt;/h1&gt;
&lt;/div&gt;</code></pre>

<h3 id="3-css-선택자-활용-계층-구조-및-속성-선택자">3. CSS 선택자 활용 (계층 구조 및 속성 선택자)</h3>
<ul>
<li><p>DOM 트리의 관계와 속성 패턴을 활용하여 요소를 정밀하게 선택할 수 있다.</p>
</li>
<li><p><strong>계층 구조 선택자</strong></p>
<ul>
<li><strong>자손 선택자 (공백)</strong>: 특정 요소 내부의 모든 해당 요소를 선택.</li>
<li><strong>자식 선택자 (<code>&gt;</code>)</strong>: 직계 자식 요소만 선택.</li>
<li><strong>인접 형제 선택자 (<code>+</code>)</strong>: 바로 뒤에 오는 첫 번째 형제만 선택.</li>
<li><strong>일반 형제 선택자 (<code>~</code>)</strong>: 뒤에 오는 모든 형제 요소를 선택.</li>
</ul>
</li>
<li><p><strong>속성 선택자 (Attribute Selectors)</strong></p>
<ul>
<li><code>[attr^=&quot;val&quot;]</code>: 지정한 값으로 <strong>시작</strong>하는 요소. (예: <code>a[href^=&quot;https&quot;]</code>)</li>
<li><code>[attr$=&quot;val&quot;]</code>: 지정한 값으로 <strong>끝나는</strong> 요소.</li>
<li><code>[attr*=&quot;val&quot;]</code>: 지정한 값을 <strong>포함</strong>하는 요소.</li>
</ul>
</li>
</ul>
<hr />
<h3 id="4-css-박스-모델box-model-및-요소-유형">4. CSS 박스 모델(Box Model) 및 요소 유형</h3>
<ul>
<li><p>모든 HTML 요소는 박스로 처리된다.</p>
</li>
<li><p><strong>요소 유형</strong></p>
<ul>
<li><strong>블록 레벨(Block)</strong>: 전체 너비를 사용하며 줄바꿈 발생. width/height 지정 가능.</li>
<li><strong>인라인 레벨(Inline)</strong>: 콘텐츠만큼의 너비만 사용. <strong>명시적인 width/height 지정이 무시됨.</strong></li>
</ul>
</li>
<li><p><strong>Width(너비)와 Height(높이)의 세부 원리</strong></p>
<ul>
<li><code>width: n%</code>: 가장 가까운 부모 컨테이너의 너비를 기준으로 계산됨.</li>
<li><strong>Height 100% 사슬 구조</strong>: 자식의 <code>height: 100%</code>는 부모의 가용 높이에 의존함. 따라서 페이지 전체 높이를 채우려면 <code>html, body</code>부터 높이가 정의되어야 함.</li>
</ul>
</li>
<li><p><strong>박스 모델 구성 및 Box-sizing</strong></p>
<ul>
<li><strong>구성</strong>: Content, Padding(안쪽 여백), Border(테두리), Margin(바깥쪽 여백).</li>
<li><strong>여백 상쇄(Margin Collapsing)</strong>: 인접 블록 요소 간 상/하 여백 중 큰 값이 적용되는 현상.</li>
<li><strong>border-box</strong>: width 설정 시 테두리와 패딩을 포함하여 계산하는 실무 권장 방식.</li>
</ul>
</li>
</ul>
<pre><code class="language-html">&lt;style&gt;
  .box-element {
    width: 300px;
    height: 150px;       /* 명시적 높이 지정 */
    padding: 20px;       /* 안쪽 여백 */
    border: 5px solid;   /* 테두리 */
    margin: 0 auto;      /* 수평 중앙 정렬 */

    /* border-box: 전체 너비를 300px로 고정하여 계산을 직관화함 */
    box-sizing: border-box; 
  }

  html, body {
    height: 100%;        /* 자식 요소의 height: 100%를 위한 사슬 구조 생성 */
  }
&lt;/style&gt;
&lt;div class=&quot;box-element&quot;&gt;Box Model &amp; Size 실습&lt;/div&gt;</code></pre>

<h3 id="5-요소-배치-및-position-전략">5. 요소 배치 및 Position 전략</h3>
<ul>
<li><p><strong>Position 속성별 특징</strong></p>
<ul>
<li><p><strong>static</strong>: 기본값. 좌표값(<code>top</code>, <code>left</code> 등) 적용 안 됨.</p>
</li>
<li><p><strong>relative</strong>: 원래 위치 기준 상대 이동. 물리적 자리 점유 유지.</p>
</li>
<li><p><strong>absolute</strong>: 문서 대열에서 이탈하여 <strong>가장 가까운 포지셔닝된 조상</strong>을 기준으로 배치됨.</p>
<ul>
<li><p><strong>조상 중 포지션 요소가 없는 경우:</strong> 초기 컨테이닝 블록인 html을 기준으로 배치됨.</p>
</li>
<li><p><strong>조상 중 포지션 요소가 있는 경우:</strong> static이 아닌 가장 가까운 조상 기준.</p>
</li>
</ul>
</li>
<li><p><strong>fixed</strong>: 뷰포트(브라우저 창) 기준 고정. 스크롤 무시.</p>
</li>
</ul>
</li>
</ul>
<hr />
<h3 id="6-단위-전략-emrem">6. 단위 전략 (em/rem)</h3>
<ul>
<li><p>환경에 따라 유동적으로 변하는 상대 단위를 사용하여 반응형 레이아웃을 설계한다.</p>
</li>
<li><p><strong>상대 단위 비교</strong></p>
<ul>
<li><strong>vw/vh</strong>: 뷰포트 너비/높이 기준 비율. 항상 화면을 참조함.</li>
<li><strong>max-width</strong>: 요소가 특정 크기 이상으로 커지는 것을 방지(특히 이미지 제어에 필수).</li>
<li><strong>em vs rem</strong>: 상속 여부에 따른 일관성 차이.</li>
</ul>
</li>
</ul>
<pre><code class="language-html">&lt;style&gt;
  html { font-size: 10px; } /* 루트 기준 */

  /* em: 부모 폰트 크기 기준 (중첩 시 복리 계산 위험) */
  .em-parent { font-size: 2em; }    /* 10px * 2 = 20px */
  .em-child { font-size: 2em; }     /* 20px * 2 = 40px */

  /* rem: 항상 루트(html) 기준 (일관성 유지) */
  .rem-parent { font-size: 2rem; }   /* 10px * 2 = 20px */
  .rem-child { font-size: 2rem; }    /* 10px * 2 = 20px (루트 참조) */
&lt;/style&gt;

&lt;div class=&quot;em-container&quot;&gt;
  em 부모 (20px)
  &lt;p&gt;em 자식 (40px - 복리 적용)&lt;/p&gt;
&lt;/div&gt;

&lt;div class=&quot;rem-container&quot;&gt;
  rem 부모 (20px)
  &lt;p&gt;rem 자식 (20px - 일관성 유지)&lt;/p&gt;
&lt;/div&gt;</code></pre>

<li>공백이 포함된 폰트명은 반드시 따옴표(<code>&quot;&quot;</code>)로 감싼다.</li>
</ul>
</li>
</ul>
