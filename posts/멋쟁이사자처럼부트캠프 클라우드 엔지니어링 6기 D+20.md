<h3 id="1-pooled-데이터소스와-커넥션-풀의-원리">1. POOLED 데이터소스와 커넥션 풀의 원리</h3>
<ul>
<li><strong>커넥션 생성 비용 절감과 응답속도 향상</strong>
데이터베이스 연결은 네트워크 핸드셰이크와 인증 과정을 거치며 응답 속도를 지연시킨다. 매 요청마다 연결을 생성하는 대신, 풀(Pool)에 미리 생성된 커넥션을 대여하고 반납하는 방식으로 응답 속도를 향상시킬 수 있다.</li>
</ul>
<pre><code class="language-xml">&lt;dataSource type=&quot;POOLED&quot;&gt;
  &lt;property name=&quot;driver&quot; value=&quot;${jdbc.dirver}&quot;/&gt;
  &lt;property name=&quot;url&quot; value=&quot;${jdbc.url}&quot;/&gt;
  &lt;property name=&quot;username&quot; value=&quot;${jdbc.userid}&quot;/&gt;
  &lt;property name=&quot;password&quot; value=&quot;${jdbc.passwd}&quot;/&gt;
  &lt;/dataSource&gt;</code></pre>
<h3 id="2-타입-별칭typealias-설정의-세-가지-방법">2. 타입 별칭(TypeAlias) 설정의 세 가지 방법</h3>
<p>매퍼 XML에서 패키지명을 포함한 긴 경로를 생략하기 위해 사용한다.</p>
<ul>
<li><strong>방법 1: 개별 등록 (configuration.xml)</strong>
특정 클래스에 대해 고유한 별칭을 직접 부여한다. </li>
</ul>
<pre><code class="language-xml">&lt;typeAliases&gt;
  &lt;typeAlias type=&quot;com.dto.EmpDTO&quot; alias=&quot;EmpDTO&quot;/&gt;
&lt;/typeAliases&gt;</code></pre>
<ul>
<li><strong>방법 2: 패키지 일괄 등록 (configuration.xml)</strong>
특정 패키지 내의 모든 클래스를 자동으로 등록한다. 별칭은 클래스명이 된다.</li>
</ul>
<pre><code class="language-xml">&lt;typeAliases&gt;
  &lt;package name=&quot;com.dto&quot;/&gt;
&lt;/typeAliases&gt;</code></pre>
<ul>
<li><strong>방법 3: 어노테이션 등록 (Java Class)</strong>
클래스 선언부에 <code>@Alias</code> 어노테이션을 사용하여 자바 코드에서 직접 별칭을 관리한다.</li>
</ul>
<pre><code class="language-java">@Alias(&quot;EmpDTO&quot;) // [이론 연결] 자바 소스에서 별칭을 정의하여 XML과의 결합도를 높인다.
public class EmpDTO {
    private int empno;
    // ... 필드 및 메서드 생략
}</code></pre>
<h3 id="3-xml-특수문자-처리-cdata와-엔티티entity">3. XML 특수문자 처리: CDATA와 엔티티(Entity)</h3>
<p>XML는 <code>&lt;</code>, <code>&gt;</code>, <code>&amp;</code> 등을 태그로 인식한다. 따라서 SQL의 비교 연산자를 처리하기 위해서 두 가지 방식을 사용한다.</p>
<ul>
<li><strong>방법 1: CDATA Section</strong>
<code>&lt;![CDATA[ ... ]]&gt;</code> 구문으로 감싼 영역은 단순 텍스트로 처리한다. 비교 연산자(<code>&lt;=</code>, <code>&gt;=</code>)가 많을 때 사용한다.</li>
</ul>
<pre><code class="language-xml">&lt;select id=&quot;selectBySal&quot; resultType=&quot;EmpDTO&quot;&gt;
  SELECT * FROM emp
  WHERE sal 
  &lt;![CDATA[ &gt;= ]]&gt; #{salary} 
  &lt;/select&gt;</code></pre>
<ul>
<li><strong>방법 2: XML 엔티티(Entity) 사용</strong>
특수 기호 대신 예약된 문자 키워드를 사용한다. </li>
</ul>

<h3 id="4-다중-dml-처리와-foreach">4. 다중 DML 처리와 <code>&lt;foreach&gt;</code></h3>
<p>자바 반복문으로 여러 번 호출하는 것보다 단일 요청으로 처리하는 것으로 코드를 간결하게 정리할 수 있으며, 자원의 소비도 줄일 수 있다.</p>
<pre><code class="language-xml">&lt;insert id=&quot;insertMulti&quot; parameterType=&quot;list&quot;&gt;
  INSERT INTO emp (empno, ename, sal)
  VALUES
  &lt;!--collection은 필수 속성이며, item=&quot;&quot;으로 받아온 데이터를 사용한다.--&gt;
  &lt;foreach item=&quot;dto&quot; separator=&quot;,&quot; collection=&quot;list&quot;&gt;
    (#{dto.empno}, #{dto.ename}, #{dto.sal})
  &lt;/foreach&gt;
  &lt;/insert&gt;

&lt;delete id=&quot;deleteMulti&quot; parameterType=&quot;list&quot;&gt;
  DELETE FROM emp
  &lt;where&gt;
    &lt;!--open - foreach - close로 진행된다 --&gt;
    &lt;!--아래의 경우: empno in ( foreach ) --&gt;
    &lt;!--separator는 구분자설정, foreach로 반복되는 내용을 구분자로 구분가능하다--&gt;
    &lt;foreach item=&quot;num&quot; separator=&quot;,&quot; open=&quot;empno in (&quot; close=&quot;)&quot; collection=&quot;list&quot;&gt;
      #{num}
    &lt;/foreach&gt;
  &lt;/where&gt;
&lt;/delete&gt;</code></pre>
<h3 id="5-동적-sql-태그와-문법-오류-방어">5. 동적 SQL 태그와 문법 오류 방어</h3>
<h4 id="5-1-if-태그와-where의-자동-교정">5-1. <code>&lt;if&gt;</code> 태그와 <code>&lt;where&gt;</code>의 자동 교정</h4>
<p>단일 조건의 참 여부를 판단한다. 이때 <code>&lt;where&gt;</code> 태그와 함께 사용하면 첫 번째 조건이 필터링되어 <code>WHERE AND ...</code> 형태가 되는 경우, 선행하는 <code>AND</code>나 <code>OR</code>를 자동으로 제거하여 문법 오류를 방지한다.</p>
<pre><code class="language-xml">&lt;select id=&quot;selectIf&quot; parameterType=&quot;EmpDTO&quot; resultType=&quot;EmpDTO&quot;&gt;
  SELECT * FROM emp
  &lt;where&gt;
    &lt;if test=&quot;job != null&quot;&gt;
      job = #{job}
    &lt;/if&gt;
    &lt;if test=&quot;sal != 0&quot;&gt;
      AND sal = #{sal}
    &lt;!-- job == null 일경우 'AND sal = '이 아니라'sal = '로 필터링 된다 --&gt;
    &lt;/if&gt;
    &lt;/where&gt;
&lt;/select&gt;</code></pre>
<h4 id="5-2-choose-태그를-이용한-다중-분기">5-2. <code>&lt;choose&gt;</code> 태그를 이용한 다중 분기</h4>
<p>자바의 <code>switch-case</code>와 유사하게 작동한다. 여러 <code>&lt;when&gt;</code> 조건 중 처음으로 일치하는 하나만 선택하며, 어떤 조건도 만족하지 않으면 <code>&lt;otherwise&gt;</code> 영역을 실행한다.</p>
<pre><code class="language-xml">&lt;select id=&quot;searchEmp&quot; parameterType=&quot;EmpDTO&quot; resultType=&quot;EmpDTO&quot;&gt;
  SELECT * FROM emp
  &lt;where&gt;
    &lt;choose&gt;
      &lt;when test=&quot;ename != null&quot;&gt;ename = #{ename}&lt;/when&gt;
      &lt;when test=&quot;job != null&quot;&gt;job = #{job}&lt;/when&gt;
      &lt;otherwise&gt;sal = 0&lt;/otherwise&gt;
    &lt;/choose&gt;
    &lt;/where&gt;
&lt;/select&gt;</code></pre>
<h4 id="5-3-set-태그를-이용한-동적-update">5-3. <code>&lt;set&gt;</code> 태그를 이용한 동적 Update</h4>
<p> <code>&lt;set&gt;</code> 태그는 내부에서 생성된 SQL 조각들의 마지막에 남은 불필요한 쉼표(<code>,</code>)를 자동으로 제거한다.</p>
<pre><code class="language-xml">&lt;update id=&quot;updateIf&quot; parameterType=&quot;EmpDTO&quot;&gt;
  UPDATE emp
  &lt;set&gt;
    &lt;!-- 'sal = 0'인 경우 첫 if의 ','가 필터링 된다. ==&gt; 'ename = val'   --&gt;
    &lt;if test=&quot;ename != null&quot;&gt;ename = #{ename},&lt;/if&gt;
    &lt;if test=&quot;sal != 0&quot;&gt;sal = #{sal}&lt;/if&gt;
  &lt;/set&gt;
  WHERE empno = #{empno}
  &lt;/update&gt;</code></pre>
<h3 id="6-트랜잭션-관리와-명시적-commit">6. 트랜잭션 관리와 명시적 Commit</h3>
<p>MyBatis의 <code>SqlSession</code>은 기본적으로 오토커밋이 비활성화되어 있다. <code>insert</code>, <code>update</code>, <code>delete</code>와 같은 DML 작업 후에는 반드시 <code>session.commit()</code>을 호출해야 데이터베이스에 물리적으로 최종 반영된다.</p>
<pre><code class="language-java">// [이론 연결] 데이터 무결성을 위해 DML 작업 성공 시 명시적으로 트랜잭션을 확정해야 한다.
try (SqlSession session = MySqlSessionFactory.getSession()) {
    int count = session.insert(&quot;insertMulti&quot;, list);
    if (count &gt; 0) {
        session.commit(); // 누락 시 데이터가 실제로 저장되지 않음
    }
}</code></pre>
